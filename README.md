# 生活キャッシュフローシミュレータ

老後やFIRE後のキャッシュフローを、公的年金、iDeCo、特定口座、税、社会保険料まで含めて計算する静的アプリです。

## 使い方

1. `index.html` をブラウザで開きます。
2. 前提条件を入力します。
3. `シミュレーションする` を押すと、比較カードと年次テーブルを計算します。
4. `初期値に戻す` で既定値へ戻せます。

## モデル化しているもの

- 公的年金と iDeCo 年金を、公的年金等控除の対象として合算
- iDeCo 拠出を、第1号被保険者・任意加入被保険者の既定上限（年81.6万円）で指定した終了年齢まで継続し、現金支出と所得控除へ簡易反映
- iDeCo 一時金を、退職所得控除を使う退職所得として概算
- 特定口座の譲渡益税を、所得税 15.315% と住民税 5% に分けて計上
- 国民健康保険料を墨田区、後期高齢者医療保険料を東京都広域連合の料率で簡易試算
- 65歳以上の介護保険料を、単身前提で墨田区の前年所得ベース段階表から概算
- 国民年金保険料を日本年金機構の月額保険料から自動計算
- 収入と取り崩しを現金に入れ、生活費・税・保険料を現金から支払うキャッシュフロー
- iDeCo の受給開始年齢と年金受取年数を戦略ごとに自動探索
- iDeCo 年金は毎年定額の受取額を候補から自動探索
- `申告する` を選ぶと、墨田区前提として譲渡益を社会保険料算定へ反映

## まだ簡略化している点

- 同一年の退職金や企業年金との控除通算
- 自治体別の国保料率や軽減判定の切り替え
- 住民税申告方法による国保算定差
- 配当所得や譲渡損失繰越控除
- 配偶者控除、扶養、医療費控除などの個別事情

## 設定ファイル

固定値として使っている初期値と保険料率は [config.json](/Users/yusuk/OneDrive/ドキュメント/codex/fire-cashflow/config.json) に切り出しています。
`index.html` をローカルサーバー経由で開ける環境では `config.json` を読み込み、読めない場合は `app.js` のフォールバック値で動きます。

## GitHub Pages

`.github/workflows/pages.yml` を追加済みです。GitHub 側で `Settings -> Pages -> Build and deployment` の `Source` を `GitHub Actions` にすると、`master` への push ごとに静的サイトを自動公開できます。

公開後の URL 例:

- `https://yskmjp.github.io/codex-fire-cashflow/`
- [GitHub Pages 公開URL](https://yskmjp.github.io/codex-fire-cashflow/)

補足:

- 公開対象はリポジトリ直下の静的ファイル一式
- `config2.json` は `.gitignore` 対象のため公開対象外
- 初回は GitHub 側で Pages を有効化するまでデプロイされません
